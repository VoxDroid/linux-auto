# Support

Thank you for using the **Linux Auto-Installation Scripts** repository! If you need help or have questions, please follow these steps:

## Getting Help
1. **Check the Documentation**:
   - Review the main [README.md](README.md) for general usage and troubleshooting.
   - Check distribution-specific README files (e.g., [Arch/README.md](Arch/README.md)) for platform-specific guidance.
   - Review script comments and inline documentation for specific script details.

2. **Search Existing Issues**:
   - Browse the [issue tracker](https://github.com/VoxDroid/linux-auto/issues) to see if your question has already been addressed.

3. **Open a New Issue**:
   - If you can't find an answer, create a new issue using the appropriate issue template in `.github/ISSUE_TEMPLATE/`.
   - Provide details about the script, your Linux distribution and version, and the issue.
   - Include any error messages, script output, or unexpected behavior.

## Common Issues and Solutions

### Script Execution Issues
- **Permission Denied**: Ensure you're running scripts with appropriate privileges (`sudo` for installation scripts).
- **Package Not Found**: Verify your distribution and package manager configuration.
- **Network Issues**: Check internet connectivity and repository mirrors.

### Distribution-Specific Help
- **Arch Linux**: Ensure pacman is up-to-date and AUR helpers are configured.
- **Debian/Ubuntu**: Check apt sources and keyrings.
- **Fedora/Red Hat**: Verify dnf configuration and RPM Fusion setup.
- **Void Linux**: Confirm xbps repositories and glibc/musl compatibility.

## Community Support
- Join discussions in the [GitHub Discussions](https://github.com/VoxDroid/linux-auto/discussions) section for community help.
- Avoid contacting maintainers directly for support; use public channels to benefit the community.

## Reporting Bugs or Suggesting Features
- For bugs, use the `bug_report.yaml` template.
- For feature requests or new scripts, use the `feature_request.yaml` template.
- For documentation improvements, use the `documentation_improvement.yaml` template.

## Security Issues
For security-related concerns, please follow the guidelines in [SECURITY.md](SECURITY.md).

We aim to respond to support requests within 48 hours. Thank you for your patience and for contributing to the project!
